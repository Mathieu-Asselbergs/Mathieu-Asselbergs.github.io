Repeating code the smart way with loops

All forms of loops use conditions in some way or another. If you are unfamiliar with conditions or would like to know more before you learn about loops, consider reading about them on this page.


Let's consider you work at NASA, or the ESA, or Spacex, and your supervisor gave you the most important task of them all: developing the countdown mechanism for the launch of their newest spaceship. And now let's consider that you, for some reason, do not feel like writing 600 printf calls for the ten minute timer. Let's consider you'd rather use loops...

for-loops: knowing when to stop

There are two main types of loops in C, the first of which is useful when you know exactly how often you want to repeat your code: the for-loop. Let me walk you through how such a loop is constructed.


Copied!

#include <stdio.h>

int main() {
    for (int i = 5; i > 0; i--) {
        printf("Just %d more minutes before launch!\n", i);
    }
}

Much like with if-statements we have a keyword, followed by something in parentheses and then a code block. This time around, the stuff in parentheses is actually three things:

  • int i = 5

    Every loop can run based on loop-specific variables. In this case we make a variable i with the value at which we start our countdown. In case you do not need any loop-specific variables, you can choose not to write anything here. The semicolon after this part is still necessary though.


  • i > 0

    Loops should end. And this second term in parentheses makes sure they do. This second part is a condition that is checked every time before the loop's code block is executed. This condition can contain the loop's specific variable and outside variables. This field can also be left empty. Doing so substitutes 1 as condition, which is always true. The semicolon after this part is necessary.


  • i--

    This update is run every time the loop's code block has been fully executed. This can be useful to update (loop-specific) variables and eventually end the loop. In this specific case, i is decremented by 1 after running through the loop once. This updates loop-specific variable i to count down until its value is low enough to no longer meet the condition of the loop. In case this part is left empty, nothing will happen after the loop's code block is done executing.


Although this type of loop is common, you can substitute any kind of variable definition, loop condition and update in a for loop. You may, for example, write a program to flex on you by printing half the alphabet backwards even though you can't:


Copied!

#include <stdio.h>

int main() {
    for (char letter = 'z'; letter != 'm'; letter--) {
        printf("%c", letter);
    }
    
    printf("\n");
}

Here, you may think that the loop would print the alphabet backwards and skip the 'm'. However, the loop stops its execution entirely whenever its condition is violated.
Crazier loops are imaginable and definitely possible! However, they all follow this general shape.


Just a dumber for-loop: the while-loop

With a detailed understanding of for-loops in place, while-loops (and do/while-loops much the same) are incredibly simple. These loops are just for-loops for which you only specify a condition and no variables or updates.
You can also think of it as an if-statement that loops back to itself when its done executing its code. In writing, these two are much alike as well: a keyword, a condition in parentheses and a code block after those parentheses:


Copied!

#include <stdio.h>

int main() {
    int init;
    printf("At what value should the countdown begin? ");
    scanf(" %d", &init);
        
    while (init > 0) {
        printf("Launching in %d seconds!\n", init);
        init -= 1;
    }
}

This example shows the syntax (how to write) of a while-loop, as well as its main strength. while-loops do not run for a fixed number of times like for-loops do. This gives flexibility that a for-loop cannot. In this example, we can take any initial countdown value as input and have a working countdown. This makes a far more tolerant and adaptive program than the one before.


The last important thing about regular while-loops is that they check their condition before running the code block.
There exists a cousin, the do/while-loop that instead checks the condition after executing the code block. This means that the code in this loop's code block is executed at least once as no condition is checked before the first iteration of the loop. Its syntax is as follows:


Copied!

#include <stdio.h>

int main() {
    int init;
    
    do {
        printf("At what value should the countdown begin? ");
        scanf(" %d", &init);
    } while (init < 1);

    while (init > 0) {
        printf("Launching in %d seconds!\n", init);
        init -= 1;
    }
}

Pay attention to what keywords are in use where and the fact that there is a semicolon after the loop's condition.

Manipulating loop behavior with continue and break

One final point about loops is that you can influence how they run with the continue and break keywords.

  • continue

    Whenever this keyword is called, the current execution of the loop's code block is ended. Execution is moved to the very end of the code block, meaning that for while and do/while-loops the condition is immediately checked. For for-loops, this means the update is immediately executed and the condition is checked. If the checked condition is satisfied, the loop starts a new execution of its code block.


  • break

    Much like continue, this keyword stops the execution of the loop's code block. However, in this case this also means that the loop is done looping. Regardless of whether the loop condition is still met or not, break ends any loop.
